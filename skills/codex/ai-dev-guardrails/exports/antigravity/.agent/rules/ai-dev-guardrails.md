# AI Dev Guardrails（Always-on）

1. 開始任何任務前：先找並遵守 `AGENTS.md` / `CONTRIBUTING.md` / `README.md`。
2. 多步驟任務必須維護 `docs/current_task.md`（Objective/Plan/Context/Handoff）。
3. 任何有意義的修改都必須寫入 `docs/ai_journal/changes.jsonl`（What/Why/Verify），並盡可能保存 patch snapshot 到 `docs/ai_journal/patches/`。
4. 不要把 token、密碼、個資寫進 journal 或 patch。
5. 預設用繁體中文溝通；必要時可中英混用。
