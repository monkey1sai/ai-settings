# Skill Evolution Loop

## 每次解決後必做（預設不另外詢問）

1. 產出一則「經驗條目」（依照 `references/capture-template.md`）。
2. 將該條目以「最新在最上方」方式追加到 `references/lessons.md`。
3. 若本次解法需要「可重複、低失誤」的自動化（例如固定指令序列、固定檔案編輯、固定資料轉換），將步驟固化成：
   - 更新既有 skill（優先），或
   - 新建一個更聚焦的全域 skill（必要時）

## 何時要新建（或拆分）一個技能

只有在符合任一條件時才新建 skill，避免技能爆量：

- 同類問題預期會重複出現（或已出現 ≥ 2 次）
- 需要穩定、可重跑的腳本（降低手動操作失誤）
- 涉及一組可通用的工作流（例如固定的除錯流程、固定的設定修正、固定的資料處理管線）

否則只更新 `references/lessons.md` 當作「經驗庫」即可。

## 建議的沉澱內容（寫進 lessons）

- **觸發語**：使用者實際怎麼問、或錯誤訊息關鍵字
- **症狀**：錯誤訊息、非預期行為、重現條件
- **根因**：真正導致問題的核心原因
- **修復**：最少步驟、可回復、可驗證
- **驗證**：如何確認已修好（含指令/檔案/截圖點）
- **防再犯**：規則、檢查清單、或需要新增的 guardrail/自動化

## 與其他規範的整合

- 需要新增/更新技能時，沿用 `skill-creator` 的規範（命名、frontmatter、內容精簡、progressive disclosure）。
- 需要執行 Python 腳本或測試時，遵守環境規則：使用 `uv` + `.venv/`，並以 Windows `.venv\\Scripts\\python.exe` 執行。
